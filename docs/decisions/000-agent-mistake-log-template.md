# 에이전트 실수 기록 로그

> 에이전트가 동일한 실수를 반복할 때, 이 파일에 기록하고 하네스에 규칙을 추가한다.
> 기록이 쌓일수록 하네스는 강해진다.

---

## 실수 기록 형식

새 실수를 기록할 때는 아래 템플릿을 복사하여 추가하라.

```
### [MISTAKE-NNN] {한 줄 제목}

**발생일**: YYYY-MM-DD  
**담당자**: {기록한 팀원 이름}  
**반복 횟수**: N회

**문제 설명**  
에이전트가 무엇을 잘못했는가? 구체적인 코드나 행동을 설명하라.

**근본 원인**  
왜 이 실수가 발생했는가? (규칙이 없었음 / 규칙이 불명확했음 / 에이전트가 문서를 읽지 않음 등)

**하네스에 추가된 해결책**  
- [ ] AGENTS.md에 금지 행동 추가 (라인 NNN)
- [ ] docs/conventions.md 규칙 보완
- [ ] ArchUnit 규칙 추가 (ArchitectureTest.java)
- [ ] Checkstyle 규칙 추가
- [ ] PR 템플릿 체크리스트 추가
- [ ] 새 ADR: docs/decisions/NNN-*.md

**검증 방법**  
이 규칙이 실제로 강제되고 있음을 어떻게 확인하는가?
```

## 사용 방법

1. 에이전트가 같은 실수를 **2회 이상** 반복하면 이 파일에 기록한다
2. 기록 즉시 하네스(AGENTS.md, docs/, 테스트)에 규칙을 추가한다
3. 주간 하네스 리뷰(`docs/decisions/weekly-harness-review.md`) 시 이 목록을 검토한다
4. 해결된 항목은 "해결" 컬럼에 ✅를 표시한다
